# Contributing to home-sustainability

Thanks for your interest in contributing! **home-sustainability** helps people track and reduce greenhouse gas emissions at home, and this project is community-driven — we welcome contributions of all kinds: code, documentation, bug reports, design feedback, and ideas.

## How to Report Bugs
- Search existing Issues first to avoid duplicates.
- If none exists, open a new Issue with:
  - A clear, descriptive title
  - Steps to reproduce the problem
  - What you expected to happen vs. what actually happened
  - Screenshots or error logs, if applicable

## How to Suggest Features
- Open an Issue labeled `enhancement`.
- Describe the problem your feature would solve, not just the feature itself (e.g. "Users can't track emissions from home heating" rather than just "Add heating tracker").
- Be open to discussion — the best features are often refined through conversation with maintainers and other contributors.

## How to Claim an Issue
- Comment on the Issue you'd like to work on so others know it's taken.
- Wait for a maintainer to confirm or assign it to you before starting significant work.
- If an Issue has been claimed but shows no activity for a couple of weeks, feel free to ask if it's still being worked on.

## How to Submit a Pull Request
1. Fork the repository.
2. Create a new branch for your change (`git checkout -b fix/short-description`).
3. Make your changes, keeping commits focused and descriptive.
4. Reference the related Issue number in your PR description (e.g. "Closes #12").
5. Open the PR against the `main` branch and fill out the PR template if one is provided.
6. Be responsive to review feedback — small follow-up commits are expected and welcome.

## Expectations for Respectful Collaboration
- Be kind and constructive in comments and reviews.
- Assume good intent; ask clarifying questions before pushing back.
- Harassment, discrimination, or disrespectful conduct will not be tolerated.

## Coding & Documentation Style
- Follow the existing code style and structure found in the repository.
- Write clear commit messages and comments where logic isn't self-explanatory.
- Update documentation (README, docstrings, etc.) alongside any relevant code changes.
- When adding emissions-related calculations or data sources, cite where the figures come from (e.g. EPA, IPCC) so users can trust the numbers.

## Getting Help
If you're stuck or unsure where to start, open an Issue labeled `question` or check for issues labeled `good first issue`.

We're glad to have you here — happy contributing to a more sustainable home!
